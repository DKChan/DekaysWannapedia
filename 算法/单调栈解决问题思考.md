# 单调栈解决问题的思考

## 背景
之前比赛时候只知道最裸的使用方法, 但是在接触更多的场景问题后, 发现有部分共性

## 问题共性
答案只要求**子序列**, 元素可以不连续, 但是前后关系要保持
同时要保持 **序列前后关系** 与 **序列内元素大小关系**
序列不可变

遇到的问题中, 会有出现区间序列的元素上下界查找的问题
递增栈(栈顶->栈底逐渐变大, 就是当前元素比栈顶大时进行出栈操作); 可以维护(n,+无穷]的下界
反之,递减栈, 维护[-无穷, n)的上界
有部分问题可以通过元素值排序并加上序号, 进行查找; 变成(val1, idx1), (val2, idx2), ..., (valn, idxn)

一般来说直接使用原序(序号序), 无法求得保证序号序情况下, 元素序的上下界
按元素序排序后, 可以直接找到最近元素, 才能保证元素序的上下界

我的理解:
1. 原序中, 按原序遍历, 单调栈保证元素序的顺序, 首次满足或者最后满足可以保证序号序的上下界
2. 元素序排序后, 元素序的遍历, 单调栈保证序号序顺序, 首次满足或者最后满足就是元素序的上下界
3. 至于是上界还是下界, 根据排序结果来, 比如元素的正序倒序, 原数组的正序遍历,倒序遍历

```fake golang

		monoStack.pop()
	}
	monoStack.append(i)
}
```
