# 特征工程与 Embedding：从声纹到行为风控

## 一、声纹特征提取的演进

声纹领域已经从传统特征工程全面转向 Embedding 方法。

| 阶段 | 技术 | 核心原理 | 现状 |
|------|------|----------|------|
| 传统特征工程巅峰 | **i-vector** | 基于联合因子分析（JFA），将高维语音统计量映射到低维"总变异空间"向量 | 2015年前主流，现已基本淘汰 |
| 早期 Embedding | **d-vector** | DNN 全连接网络提取每帧隐层输出，取平均作为声纹向量 | 极少直接使用 |
| 里程碑 Embedding | **x-vector** | TDNN（时延神经网络）捕捉时序上下文 + 统计池化层（Statistics Pooling）提取均值和方差 | 经典基线，奠定现代声纹基础 |
| 工业界顶流 | **ECAPA-TDNN** | TDNN + ResNet 残差连接 + 多尺度特征聚合 + SE 注意力机制 | 落地最广泛、性价比最高 |
| 高精度方案 | **ResNet / CAM++ / ERResNet** | 图像领域 ResNet 魔改，处理二维 Fbank 特征图 | 比赛夺冠常客，计算量较大 |

**关键洞察：** 声纹的本质流程是 `语音 → Fbank 特征图 → 深度网络 → Embedding 向量`，传统手工特征工程已被端到端的 Embedding 方法取代。

## 二、行为序列 Embedding 化（行为风控）

核心类比：**用户连续的行为流水 = 互联网世界里用户的"声音"**

行为（点击、滑动、下单、提现）类比为"单词"，通过 Embedding 技术将稀疏、高维、非线性的行为流压缩为低维稠密向量。

### 路径一：行为当做"单词"（Behavior-to-Vector）

借鉴 NLP 思想，最直接的量化方式。

**Step 1 - 行为离散化与编码（Tokenization）**

```
行为 = 动作 + 资产/页面
例："点击_发送验证码"、"页面进入_贷款申请页"、"修改_银行卡"
```

构建词表（Vocabulary），枚举全站核心行为。

**Step 2 - 序列化（Sequence Generation）**

按时间顺序串联用户行为：

```
[进入首页, 点击借款, 输入金额, 点击发送验证码, 绑定银行卡, 提交审核]
→ 对应词表 ID → [12, 45, 8, 102, 67, 89]
```

**Step 3 - 训练提取 Embedding**

- **方法 A（轻量级）：Word2Vec / Item2Vec** — 无监督，纯看行为上下文共现关系
- **方法 B（端到端）：LSTM / GRU / Transformer** — 接风控业务目标训练，倒数第二层输出即为序列 Embedding

### 路径二：多维属性融合量化（Multi-Modal Embedding）

单纯"动作"不够，需融合时间、环境、频率等维度。

**单条日志的复合量化：**

```
E_total_log = E_action + E_time + E_context
```

- **动作（Action）：** 映射为 Embedding
- **时间间隔（Time Delta）：** 连续值分桶离散化（<1s, 1s-5s, >5s），过时间 Embedding
- **环境/设备（Context）：** 网络环境、设备 ID 变化等

每条日志变成复合向量后，整个行为流水变成矩阵（行=时间步，列=特征维度），**类似声纹的 Fbank 特征图**，可直接送入 TextCNN 或 Transformer。

### 路径三：图特征表征（Graph Embedding）

适用于黑产关联分析（频繁换设备、共享 IP、批量操作）。

- **量化方式：** 用户/设备/IP 作为不同类型节点，行为作为边
- **提取 Embedding：** GCN / GraphSAGE，通过邻居信息聚合生成 Graph Embedding
- **效果：** 正常账号若邻居大量连着黑产设备，其 Embedding 会向黑产聚拢

## 三、工业落地实用建议

| 问题 | 解决方案 |
|------|----------|
| 行为长度差异极大 | **定长截断与填充**：设定最大长度（如最近 50-100 个行为），超过截断，不足用 [PAD] 补齐 |
| 长序列噪声 | **Session / 滑动窗口切分**：按 Session（半小时无操作断开）或滑动时间窗口（如最近 10 分钟）切分 |
| 新功能/页面上线 | **OOV 兜底**：保留 [UNK] 标记位处理未知操作 |
| 可解释性差 | **混合架构**：传统专家特征（近1小时点击次数、IP变更等）+ 序列 Embedding 特征，一起输入 XGBoost 等模型 |

## 四、核心思想总结

```
声纹：  语音波形 → Fbank 特征图 → ECAPA-TDNN → Speaker Embedding
行为风控：用户行为流水 → 多维量化矩阵 → Transformer/LSTM → Behavior Embedding
```

两者的本质逻辑相通：**将原始的时序信号，通过前置量化转为结构化表征，再用深度网络压缩为低维稠密向量，用于下游任务（识别/分类/聚类）。**
