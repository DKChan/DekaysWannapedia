### 网络威胁简介

#### 恶意程序

- **攻击载荷**

  发起攻击能建立网络连接的载体, 像是攻击向量的中最前置的手段

  例如: 下图的各种方式

  ```
                      [钓鱼邮件]
                         ↑
                         |
      [远程攻击类载荷] ← [攻击者] → [恶意文档]
           ↗    ↕    ↖
     [WebShell]  [标准Shell]
           ↘    ↓    ↙
                ↓
           [反弹Shell]
                ↓
           [后门木马]

  图1-1 常见的攻击载荷

  说明:
  ← 向外投放: 攻击者主动发起的攻击向量
  → 向内回连: 成功入侵后建立的连接通道
  ↕ 双向交互: 标准Shell具备完整双向通信能力
  ```


- **木马**

  相比前面直接到达你服务器/PC, 通过传递木马文件, 触发攻击; 但要达到传送还是需要前者的建立吧?

  ```
                盗号木马(窃取账号)
                       ↑
                       |
     主页木马 ←——— 🐴木马中心 ———→ 勒索木马
   (锁浏览器)              (加密文件)
                       |
     网银木马 ←——— | ———→ 挖矿木马
   (盗取钱财)       |      (挖数字货币)
                       |
           后门木马 ↘ ↙ 广告木马
           (窃取资料)  (乱弹广告)

  图1-2 木马的种类
  ```


- **蠕虫**

  最大特性是: 自我复制, 主动传播;

  这种自主性和蔓延性很像爬虫, 一个是拉取信息, 一个传播信息

  ```
                      蠕虫病毒（总类）
                           │
          ┌────────────────┼────────────────┐
          │                │                │
      网络传播          邮件传播          即时通讯      文件共享
          │                │                │            │
          ↓                ↓                ↓            ↓
      [网络蠕虫]       [邮件蠕虫]       [聊天蠕虫]   [共享蠕虫]

  图1-3 蠕虫的分类（按传播途径）
  ```


- **感染型病毒**

  前面2,3都算是广义的病毒, 狭义的病毒不停留在应用/软件层次

  ```
  ┌─────────────────────────────────────────┐
  │                                         │
  │   Ring3: 应用软件层（木马、蠕虫、感染型）  │  ← 用户态（权限最低）
  │                                         │
  ├─────────────────────────────────────────┤
  │                                         │
  │   Ring0: 系统内核层（Rootkit病毒）        │  ← 内核态（高权限）
  │                                         │
  ├─────────────────────────────────────────┤
  │                                         │
  │   Boot: 硬件加载层（Bootkit病毒）         │  ← 启动层（权限最高）
  │                                         │
  └─────────────────────────────────────────┘
          ↑ 越往下，权限越高，越难清除

  图1-4 恶意软件权限层级
  ```

### 安全运营简介

#### 安全运营定义

发现威胁->分析威胁->处理威胁|共享情报

#### 发展阶段

#### 恶意程序对抗

1. 躲避技术

书中列举8种方式, 不赘述了, 主要是都为了隐藏自身和自身特征, 通过可见/混淆/寄生等等方式

2. 破坏技术

针对查杀工具的破坏, 使发现者瘫痪

#### 云查杀

只是集中式的信息数据整合/更新, 还是正常的查找匹配手段

#### 大数据应用

感知的发展阶段

1. 用户反馈: 滞后且影响面大
2. 规则探知: 容易被对抗绕过
3. 大数据挖掘: 其实还是有很多扩展方向, 提高识别速度和广度
